# ConvoPro — Private ChatGPT Clone

A locally hosted, private AI chat application built with Streamlit, Ollama, and MongoDB.  
It gives you a ChatGPT-style interface where all your conversations are stored in your own database, and all AI inference runs on your own machine — no data leaves your system.

---

## What It Achieves

- **Private AI chat** — runs LLMs locally via Ollama (e.g. `gemma2:2b`)
- **Multi-model support** — switch between any Ollama models from a dropdown
- **Persistent conversations** — all chats are saved in MongoDB and survive page refreshes
- **Auto-generated titles** — each new conversation gets a meaningful title generated by the LLM
- **Sidebar history** — browse, switch, and start new conversations like ChatGPT

---

## Tech Stack

| Layer        | Technology                        |
|--------------|-----------------------------------|
| UI           | Streamlit                         |
| LLM Inference| Ollama + LlamaIndex               |
| Database     | MongoDB (via PyMongo)             |
| Config       | Pydantic Settings + python-dotenv |

## Methods & Their Functionality

### `config/settings.py` — `Settings`
Reads and validates required environment variables using Pydantic.  
Falls back to a `.env` file if variables aren't set at the OS level.

| Variable | Purpose |
|---|---|
| `MONGO_DB_URL` | MongoDB connection string |
| `MONGO_DB_NAME` | Database name |
| `OLLAMA_URL` | Ollama server URL |
| `OLLAMA_MODELS` | Comma-separated list of available models |

---

### `db/mongo.py` — `get_collection(name)`
Opens a MongoDB connection using settings and returns a reference to the named collection.

---

### `db/conversations.py`

| Method | What it does |
|---|---|
| `create_new_conversation(title, role, content)` | Creates a new conversation document in MongoDB with an optional first message |
| `add_message(conv_id, role, content)` | Appends a new message to an existing conversation and updates its timestamp |
| `get_conversation(conv_id)` | Fetches a full conversation document by ID and refreshes its `last_interacted` timestamp |
| `get_all_conversations()` | Returns all conversations as `{id: title}`, sorted by most recently active |

---

### `llm_factory/get_llm.py` — `get_ollama_llm(model_name)`
Creates an Ollama LLM instance for the given model name.  
Caches the last used instance — if the same model is requested again, it reuses the existing object instead of creating a new one.

---

### `services/chat_utilities.py` — `get_answer(model_name, chat_history)`
Sends the full conversation history (with a system prompt) to the LLM and returns the assistant's reply.  
Uses LlamaIndex's `ChatMessage` to properly format `user` and `assistant` turns.

---

### `services/get_title.py` — `get_title(model, user_query)`
Prompts the LLM to generate a short (max 7 words) title for a conversation based on the user's first message.  
Uses LlamaIndex's `PromptTemplate` to structure the prompt.

---

### `services/get_models_list.py` — `get_ollama_models_list()`
Reads the `OLLAMA_MODELS` config value (a comma-separated string) and returns it as a clean Python list.

## Setup

1. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

2. **Create a `.env` file** using `env_template.txt` as reference:
   ```env
   MONGO_DB_URL="mongodb://localhost:27017/"
   MONGO_DB_NAME="chat_app"
   OLLAMA_URL="http://localhost:11434"
   OLLAMA_MODELS="gemma2:2b"
   ```
   > For multiple models: `OLLAMA_MODELS="gemma2:2b, llama3:8b"`

3. **Start Ollama** and pull your model:
   ```bash
   ollama pull gemma2:2b
   ```

4. **Run the app**
   ```bash
   streamlit run main.py
   ```

## How It Works (Flow)

```
User types a message
       │
       ├─► [New conversation?] ──► get_title() ──► create_new_conversation()
       │
       ├─► add_message()  (saves user message to MongoDB)
       │
       ├─► get_answer()   (sends full history to Ollama via LlamaIndex)
       │
       └─► add_message()  (saves assistant reply to MongoDB)
```
